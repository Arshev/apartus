# Адаптированные промпты ревью и генерации

Копилка для «Продвинутого/Отличного» уровня сдачи. Сюда складываем промпты,
которые после правок стабильно дают хороший результат в нашем проекте Apartus.

## Базовые промпты (из sdd-workflow.md, как отправная точка)

### Brief review (базовый)

```
Ты — ревьюер бизнес-задач. Проверь этот Brief на полноту и однозначность.

Критерии:
1. Проблема конкретна и измерима (не «улучшить», а конкретная метрика или боль)
2. Назван стейкхолдер или пользователь, для которого решаем
3. Понятен контекст: откуда задача, почему важна сейчас
4. Brief НЕ содержит решения — только проблему и желаемый результат
5. Нет двусмысленных формулировок («быстро», «удобно», «при необходимости»)

Для каждого найденного замечания укажи:
- Что именно не так (цитата из документа)
- Почему это проблема
- Как исправить (конкретное предложение)

Если замечаний нет — напиши «0 замечаний, Brief готов к работе».
```

### Spec review (базовый, TAUS)

```
Ты — строгий ревьюер спецификаций для AI-агентов. Проверь эту спеку по критериям TAUS.

Для каждого критерия дай оценку pass/fail:

1. Testable — есть ли конкретные acceptance criteria, по которым можно написать автотест?
2. Ambiguous-free — есть ли двусмысленные слова («быстро», «удобно», «при необходимости», «и т.д.»)?
3. Uniform — описаны ли все состояния (loading, error, success, empty)? Все сценарии ошибок?
4. Scoped — это одна фича? Меньше 1500 слов? Не затрагивает больше 3 модулей?

Дополнительно проверь:
5. Есть ли ссылка на Brief/Issue?
6. Указан ли scope (что входит И что НЕ входит)?
7. Перечислены ли инварианты?
8. Указаны ли ограничения на реализацию?

Для каждого fail:
- Цитата из спеки
- Почему это проблема для агента
- Конкретное предложение по исправлению

Если все критерии pass — напиши «0 замечаний, спека готова к реализации».
```

### Plan review (базовый)

```
Ты — ревьюер планов реализации. Проверь этот план на выполнимость и корректность.

Критерии:
1. Каждый шаг конкретен — понятно, какой файл затрагивается и что именно меняется
2. Порядок шагов корректен — зависимости не нарушены, нет циклов
3. Каждый шаг атомарен — можно выполнить и проверить независимо
4. Нет пропущенных шагов — миграции, тесты, документация
5. Grounding: упомянутые файлы и модули существуют в проекте

Для каждого найденного замечания:
- Какой шаг затронут
- Почему это проблема
- Как исправить

Если замечаний нет — напиши «0 замечаний, план готов к реализации».
```
