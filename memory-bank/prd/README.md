# PRD Index

Apartus-level продуктовый контекст полностью укладывается в [`../domain/problem.md`](../domain/problem.md). PRD заведём только когда появится инициатива, которая:

- распадается на несколько feature packages с общим продуктовым upstream;
- требует фиксации users/goals/scope/metrics на уровне инициативы (не одной фичи).

Пока таких инициатив нет. Каталог остаётся пустым. Шаблон: [`../flows/templates/prd/PRD-XXX.md`](../flows/templates/prd/PRD-XXX.md).
