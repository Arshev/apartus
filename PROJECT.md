# Apartus — Property Management System

## Что это

Apartus — SaaS-система управления краткосрочной и долгосрочной арендой недвижимости (PMS). Выступает единым центром управления объектами, бронированиями, гостями и финансами, а также хабом между площадками-агрегаторами (Booking.com, Airbnb и др.) и собственниками недвижимости.

## Для кого

- **Управляющие компании (УК)** — компании, управляющие чужой недвижимостью за комиссию (10–500+ юнитов). Нуждаются в прозрачной отчётности перед собственниками, автоматическом расчёте выплат и масштабируемом инструменте.
- **Частные арендодатели** — собственники 1–10 объектов, самостоятельно управляющие арендой. Нуждаются в простом инструменте для синхронизации календарей, приёма бронирований и учёта доходов.

Оба сегмента обслуживаются через разные тарифные планы по количеству юнитов.

## Зачем

Рынок краткосрочной аренды активно растёт, но большинство арендодателей и небольших УК ведут учёт вручную или используют разрозненные инструменты. Apartus объединяет календарь, каналы продаж, ценообразование, CRM, финансы и обслуживание в одной системе с фокусом на русскоязычный рынок.

---

## Сейчас делаем (MVP)

Текущий фокус — минимально работающий PMS для частного арендодателя и небольшой УК. В MVP входят только следующие функции:

1. **Мультитенантность и доступ** — организация, пользователи, базовые роли (Owner / Manager / Member), подразделения (Branches). Авторизация через встроенный Rails 8 auth + Pundit.
2. **Объекты и юниты** — CRUD для Property и Unit с базовыми полями (адрес, тип, вместимость, статус, фото).
3. **Календарь бронирований** — ручное создание/редактирование Reservation, визуальный календарь по юнитам, статусы (new / confirmed / checked-in / checked-out / cancelled), check-in и check-out, блокировки дат.
4. **Базовое ценообразование** — цена за ночь по юниту и сезонные цены (периоды с разными ставками). Расчёт стоимости бронирования.
5. **Гости (минимум CRM)** — карточка гостя (контакты, история бронирований), связь с Reservation. Без воронок и автоматизаций.

Этого достаточно, чтобы одна организация могла вести реальные бронирования в системе от начала до конца.

## Не входит в MVP (Non-Goals / Future)

Перечисленные ниже модули из каталога возможностей **сознательно откладываются** до завершения MVP и не должны реализовываться раньше времени:

- Channel Manager и любые интеграции с Booking.com / Airbnb / Ostrovok
- Встраиваемый виджет бронирования и онлайн-оплата
- Платёжные шлюзы (ЮKassa, Stripe), выставление счетов, расходы, P&L, RevPAR/ADR/occupancy-аналитика
- Модуль собственников (комиссии, акты выплат, личный кабинет собственника)
- Полноценный CRM: воронки, лиды, LeadSource, автоматизации, таймлайн активностей
- Задачи, канбан, наряд-заказы, инспекции юнитов
- Коммуникации: внутренний мессенджер, авто-сообщения, Unified Inbox, Telegram-бот
- Динамическое ценообразование, скидки за длительность, программа лояльности
- Кастомные роли с гранулярными разрешениями (сверх трёх базовых)

Эти возможности остаются в общем каталоге модулей ниже как целевая картина продукта, но в рамках текущей фазы реализации они **out of scope**.

## Стек

### Backend

- Ruby on Rails 8 (авторизация из коробки)
- PostgreSQL
- Pundit (авторизация и кастомные роли)
- REST API (`/api/v1`)

### Frontend

- Vue.js 3 (чистый JavaScript, без TypeScript)
- Vuetify 3 (UI-фреймворк)
- Vite (сборка)
- Pinia (управление состоянием)
- Vue Router 4

### Структура проекта

Монорепо с двумя директориями:

```text
/backend   — Rails API
/frontend  — Vue.js SPA
```

### Локализация

- Система: русский (основной), английский
- Виджет бронирования: русский, английский (расширяемо)

## Модули системы

### 1. Объекты и юниты

Иерархия: **Организация → Объект (Property) → Юнит (Unit)**

- **Объект (Property)** — здание или комплекс: отель, апарт-комплекс, жилой дом, хостел. Содержит адрес, тип, описание, фото, контакты.
- **Юнит (Unit)** — конкретная единица размещения внутри объекта: номер, квартира, комната, койко-место. Содержит тип, вместимость, список удобств (amenities), фотографии, описания, статус (доступен, на обслуживании, заблокирован).

Поддерживаемые типы недвижимости:

- Квартиры и апартаменты
- Отели и гостиницы
- Дома, виллы, коттеджи
- Хостелы и койко-места

### 2. Календарь бронирований

Центральный модуль системы — визуальный календарь доступности по всем юнитам.

- Создание и управление бронированиями (Reservation/Booking)
- Статусы: новое, подтверждено, ожидает оплаты, заселён (checked-in), выселен (checked-out), отменено, no-show
- Check-in / check-out управление с фиксацией времени
- Блокировка дат (обслуживание, личное использование собственника)
- Фильтрация по объектам, типам юнитов, статусам
- Визуальное отображение источника бронирования (канал)

### 3. Channel Manager

Полноценный менеджер каналов с двусторонней синхронизацией.

- Подключение площадок: Booking.com, Airbnb, Ostrovok и др.
- Автоматическое обновление доступности на всех площадках при новом бронировании
- Синхронизация цен и ограничений (мин. срок, закрытие продаж)
- Импорт бронирований с площадок в единый календарь
- Управление контентом (описания, фото) через систему
- Мониторинг статуса синхронизации по каждому каналу

### 4. Ценообразование

Гибкая система управления ценами.

- **Базовые тарифы** — цена за ночь по каждому юниту
- **Сезонные цены** — настраиваемые периоды с разными ставками (высокий сезон, праздники, низкий сезон)
- **Скидки за длительность** — автоматические скидки при бронировании от N ночей (например: 7+ ночей = -10%, 30+ = -20%)
- **Динамическое ценообразование** — правила автоматической корректировки цен на основе загрузки и спроса
- **Программа лояльности** — накопление баллов за бронирования, скидки для постоянных гостей, уровни лояльности

### 5. Виджет бронирования

Встраиваемый модуль для сайтов собственников и УК.

- Подключение через iframe или JS-сниппет
- Настройка внешнего вида (цвета, логотип, шрифты) из панели управления
- Адаптивный дизайн (мобильные, планшеты, десктоп)
- Выбор дат, юнита, количества гостей
- Отображение актуальных цен с учётом сезонности и скидок
- Приём онлайн-оплаты через виджет
- Локализация (RU, EN)
- Подтверждение бронирования с отправкой email/SMS

### 6. Оплата и финансы

#### Способы оплаты

- **Онлайн-оплата картой** — интеграция с платёжным шлюзом (ЮKassa, Stripe и т.д.)
- **Депозит + остаток** — частичная предоплата при бронировании, остаток при заселении. Настраиваемый % депозита.
- **Оплата на месте** — наличные или терминал при заселении
- **Выставление счетов** — для долгосрочной аренды: ежемесячные счета, отслеживание задолженности

#### Учёт

- Доходы от бронирований с разбивкой по объектам, юнитам и каналам
- Расходы на обслуживание, коммуналку, ремонт — привязка к объектам/юнитам
- Кассовый учёт: наличные и безналичные операции
- P&L по объектам

#### Аналитика

- **Occupancy rate** — загрузка по объектам и юнитам
- **RevPAR** (Revenue Per Available Room) — доход на доступный юнит
- **ADR** (Average Daily Rate) — средняя цена за ночь
- Доходность по каналам бронирования
- Настраиваемые отчёты и дашборды

### 7. Управление собственниками (Owners)

Модуль для УК, управляющих чужой недвижимостью.

- Привязка объектов и юнитов к собственникам
- Настраиваемая комиссия УК (% от дохода, фиксированная ставка, комбинированная)
- Автоматический расчёт доли собственника: доходы минус комиссия УК и расходы на обслуживание
- Формирование актов выплат
- Личный кабинет собственника с доступом к отчётам по своим объектам
- История выплат и взаиморасчётов

### 8. CRM

Система управления взаимоотношениями с клиентами.

- **Воронки продаж (Pipelines)** — настраиваемые стадии для разных типов клиентов
- **Два типа клиентов:**
  - Гости (Guest) — краткосрочная аренда, туристы, командировки
  - Арендаторы (Tenant) — долгосрочная аренда, постоянные жильцы
- **Лид → Гость/Арендатор** — конверсия лида в клиента при бронировании/заключении договора
- **Карточка клиента** — контакты, история бронирований, предпочтения, документы, заметки, уровень лояльности
- **Источники лидов** (LeadSource) — откуда пришёл клиент (сайт, звонок, площадка, рекомендация)
- **История взаимодействий** — таймлайн активностей по каждому лиду/клиенту
- **Автоматизация:**
  - Триггеры: создание лида, смена стадии, прошло N дней без активности
  - Действия: назначить ответственного, отправить уведомление, создать задачу, переместить на стадию

### 9. Задачи и обслуживание

Система управления задачами и техническим обслуживанием объектов.

#### Задачи

- Канбан-доски с настраиваемыми стадиями
- Приоритеты, сроки, назначение на сотрудников
- Рекуррентные задачи (расписание уборок, плановые проверки)
- Подзадачи, метки, комментарии, вложения
- Связь с бронированиями и лидами из CRM
- Автоматическое создание задач на уборку при check-out

#### Наряд-заказы (Work Orders)

- Заявки на ремонт и обслуживание с привязкой к юниту
- Статусы: открыт, в работе, завершён
- Расчёт стоимости (материалы + работа)
- Назначение исполнителя
- Документы и фото

#### Инспекции юнитов

- Осмотр при заселении и выселении (check-in / check-out inspection)
- Чек-листы с шаблонами
- Фиксация повреждений с фото
- Автоматическое создание наряд-заказа при обнаружении проблем

### 10. Коммуникации

- **Внутренний мессенджер** — чат между сотрудниками организации, групповые беседы
- **Авто-сообщения гостям** — настраиваемые шаблоны автоматических сообщений:
  - Подтверждение бронирования
  - Инструкция по заселению (за N дней до check-in)
  - Запрос отзыва (после check-out)
  - Напоминание об оплате
- **Unified Inbox** — единый ящик входящих сообщений от гостей со всех каналов (Booking, Airbnb, WhatsApp, email) в одном интерфейсе
- **Telegram-бот** — уведомления сотрудникам: новые бронирования, задачи, оплаты, срочные заявки

### 11. Организация, подразделения и доступ

#### Мультитенантность

- Организация (Organization) — корневая сущность, изолирует данные между клиентами системы

#### Подразделения (Branches)

- Иерархическая структура с parent_id (например: «Сочи» → «Адлер», «Центр»)
- Наследование таймзоны от родительского подразделения
- Объекты (Property) привязываются к подразделениям
- Scope видимости для сотрудников: только своё подразделение, с дочерними, все

#### Роли и разрешения

- Базовые роли: Владелец (Owner), Менеджер (Manager), Сотрудник (Member)
- Кастомные роли с гранулярными разрешениями (Pundit)
- Группы разрешений по модулям: объекты, бронирования, финансы, CRM, задачи, настройки и т.д.
- Собственник недвижимости — отдельная роль с ограниченным доступом к отчётам по своим объектам

## Ключевые принципы

- **Календарь в центре** — все модули связаны через календарь бронирований
- **Мультиканальность** — система является единой точкой управления для всех каналов продаж
- **Прозрачность для собственников** — автоматические отчёты и расчёт выплат
- **Автоматизация рутины** — авто-сообщения, авто-задачи на уборку, рекуррентные проверки
- **Гибкость ценообразования** — сезонность, скидки, лояльность, динамические цены
- **Локализация** — русский как основной язык, английский как второй, расширяемо
