# Impeccable Design Context — Apartus

> Canonical design context for the Impeccable skill. Required before any design work.
> Last updated: 2026-04-17.

## Design Context

### Users

**Primary:** менеджер управляющей компании / владелец апартаментов за рабочим столом, 8+ часов в день. Инструмент должен **не мешать**: быстрое сканирование календаря, уверенный клик, минимум decorative noise. Русскоязычный рынок.

**Secondary:** владелец недвижимости, который заходит раз в неделю посмотреть «сколько заработал, что впереди» — должен мгновенно прочитать dashboard без обучения.

**Tertiary (mobile):** cleaner / field staff — смотрят задачи и check-in/out с телефона между юнитами. Адаптация UI, не амputация.

**Job-to-be-done:** вести операционку PMS без раздражения. Календарь, брони, задачи, финансы — всё в одном месте, без контекст-свитчинга между tools.

### Brand Personality — 3 слова

**Operational. Trustworthy. Quietly modern.**

- **Operational** — это ежедневный рабочий инструмент, не beautiful product page. Density > whitespace в data-heavy surfaces.
- **Trustworthy** — финансовые цифры и расписание гостей должны читаться без double-check. Conservative typography, no playful experiments на critical paths.
- **Quietly modern** — 2026, не 2018. Уверенный, немного editorial, typography-driven. НЕ кричит «я современный» через gradients и glow.

**Emotional goals:** confidence (данные верны), efficiency (быстро сделал и закрыл), calm (не утомляет за 8 часов).

### Aesthetic Direction

**References (вдохновение):**
- **Linear** — clean density, accent used sparingly, works 8 hours without fatigue
- **Notion calendar** — calendar as reading surface, not spreadsheet
- **Modern editorial dashboards** (Stripe, Pipedrive updated) — typographic hierarchy does the work

**Anti-references (что не делаем):**
- **RentProg / "бухгалтерский софт 2008"** — grey boxes, identical buttons, screaming status colors
- **"AI SaaS slop"** — cyan/purple gradients, glowing borders, Inter + DM + Space Grotesk монокультура, cards-inside-cards
- **Material Design default look** — generic Vuetify-from-the-box (наше текущее состояние)
- **Luxury / resort booking aesthetic** — мы не про красоту объектов, а про управление ими

**Theme:** Light primary (daytime operational work), dark fully supported (late-night shifts happen). Both must be first-class — не ставить dark на second-place.

**Typography:**
- Distinctive display face для headings (editorial, с характером). Банлист ниже.
- Quiet body font для data-dense tables (high legibility, subtle).
- **НЕ использовать:** Inter, Roboto, Arial, system defaults, IBM Plex, Space Grotesk/Mono, Plus Jakarta Sans, Fraunces, Newsreader, Playfair, DM Sans/Serif, Instrument Sans/Serif, Outfit, Cormorant, Crimson, Syne, Lora.
- Monospace допустим для IDs и timestamps (не для «tech vibe»).

**Color:**
- Palette via **OKLCH** (перцептуально равномерная, maintainable).
- Tinted neutrals, не pure `#fff` / `#000`. Оттенок к brand green.
- Existing green primary (`#43A047`) можно сохранить как hue anchor, но перевести в OKLCH и refine chroma by lightness.
- **Status colors стадий бронирований — функциональны, не декоративны.** Остаются saturated (blue/green/grey/red/orange) — они несут смысл (FT-020).
- **60 / 30 / 10 rule:** 60% neutral surfaces, 30% secondary text/borders, 10% accent (primary green). Один accent — скупо.

**Spacing:** 4pt scale with semantic tokens (`--space-xs..2xl`), not pixel-named. Vary spacing — density в data tables, generous в settings/empty states.

**Motion:** exponential easing (ease-out-quart), only entrances/state changes. No bounce, no elastic, no decorative micro-interactions on 8h screens.

### Design Principles

1. **Data density is a feature.** В operational screens (Gantt, reservation list, finances) плотность — благо. Не раскидывать whitespace ради «воздуха».
2. **One accent, used sparingly.** Primary green — только для CTA и active state. Не для декора, разделителей, штрихов.
3. **Typography does the work.** Hierarchy через size / weight / case, не через boxes, borders, cards.
4. **Status color ≠ decoration.** Red means a problem. Если элемент красный — пользователь должен действовать. Никогда не «красный потому что красиво».
5. **Respect the 8-hour session.** Никаких bounce-анимаций, glow, pulse на не-alert элементах. Motion — функциональный, реduce-motion respected.
6. **Localized literally.** Русский — первый язык. Все UI-копии должны быть тональностью соответствовать Apartus brand voice (trust, без подобострастия и без корпоративной сухости).

### Constraints

- **Stack:** Vue 3 + Vuetify 4 + Composition API, no TypeScript (ADR-002). Редизайн должен работать **поверх Vuetify theme system** (CSS vars `--v-theme-*`), не ломать component defaults и dark mode.
- **No new npm deps** без явного ADR (CON-01 feature template).
- **i18n parity ru/en** обязательна для любого нового текста.
- **Accessibility:** WCAG AA contrast target. Motion-reduced users — graceful degradation.
- **Coverage ratchet:** frontend tests не падают при визуальных изменениях (см. testing-policy.md).
